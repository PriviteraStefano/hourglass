CREATING ISSUES IN GITHUB
================================================================================

Option 1: Manual (GitHub Web)
  → Fastest for small counts
  → Create master, then each sub-issue
  → ~5 minutes total

Option 2: GitHub CLI (gh)
  → Good for bulk creation
  → Scriptable
  → ~10 minutes total

Option 3: GitHub Projects
  → Best UX for team
  → Automatic tracking & dependencies
  → Can create issues inline
  → ~20 minutes total
