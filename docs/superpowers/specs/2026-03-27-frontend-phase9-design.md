# Frontend Phase 9 Design: Time Entry End-to-End Slice

**Vertical slice: Auth + Employee time tracking flow**

---

## 1. Overview

This phase delivers a working frontend for the most common user journey: logging time entries. We implement authentication, the application shell, and a complete time entry experience from viewing the calendar to submitting entries for approval.

### Scope

**In scope:**
- User registration (create org + admin)
- Login/logout
- Application shell with sidebar navigation
- Time entries calendar view (hybrid: mini-calendar + detail panel)
- Create/edit/delete draft time entries
- Submit single entry or all entries for a month

**Out of scope:**
- Approvals UI (Phase 11)
- Expenses UI (Phase 10)
- Contracts/Projects management (Phase 12)
- Customer view (Phase 13)
- CSV exports (Phase 8)
- Activation flow for invited users (Phase 9.2)

## 5. Authentication

### Auth Flow

1. **Register**: POST `/auth/register` → creates org + admin user → returns JWT
2. **Login**: POST `/auth/login` → returns JWT
3. **Token Storage**: localStorage (`auth_token`)
4. **Auth Check**: GET `/auth/me` (or decode JWT locally)
5. **Logout**: Clear localStorage, redirect to `/login`

### API Client

**`lib/api.ts`**
```tsx
const API_BASE = import.meta.env.VITE_API_URL || 'http://localhost:8080'

export async function api<T>(
  path: string,
  options?: RequestInit
): Promise<T> {
  const token = localStorage.getItem('auth_token')
  
  const res = await fetch(`${API_BASE}${path}`, {
    ...options,
    headers: {
      'Content-Type': 'application/json',
      ...(token && { Authorization: `Bearer ${token}` }),
      ...options?.headers,
    },
  })

  if (res.status === 401) {
    localStorage.removeItem('auth_token')
    window.location.href = '/login'
    throw new Error('Unauthorized')
  }

  if (!res.ok) {
    const error = await res.json().catch(() => ({ message: 'Request failed' }))
    throw new Error(error.message || error.error || 'Request failed')
  }

  return res.json()
}
```

### useAuth Hook

```tsx
export function useAuth() {
  const queryClient = useQueryClient()
  
  const { data: user, isLoading } = useQuery({
    queryKey: ['auth', 'me'],
    queryFn: () => api<User>('/auth/me'),
    retry: false,
    staleTime: Infinity,
  })

  const login = useMutation({
    mutationFn: (creds: { email: string; password: string }) =>
      api<{ user: UserWithMembership; token: string }>('/auth/login', {
        method: 'POST',
        body: JSON.stringify(creds),
      }),
    onSuccess: ({ user, token }) => {
      localStorage.setItem('auth_token', token)
      queryClient.setQueryData(['auth', 'me'], user)
    },
  })

  const logout = () => {
    localStorage.removeItem('auth_token')
    queryClient.clear()
    router.navigate({ to: '/login' })
  }

  return { user, isLoading, login, logout }
}
```

## 9. Entry Detail Component

### States

1. **No entry for date**: Show "Create entry" button
2. **Draft entry**: Editable form with project rows
3. **Submitted/Pending**: Read-only view, status badge
4. **Approved**: Read-only view, approved badge
5. **Rejected**: Editable (can fix and resubmit)

### Component

```tsx
export function EntryDetail({ date, entry }: EntryDetailProps) {
  const createEntry = useCreateTimeEntry()
  const updateEntry = useUpdateTimeEntry()
  const deleteEntry = useDeleteTimeEntry()
  const submitEntry = useSubmitEntry()
  
  const form = useForm({
    defaultValues: {
      items: entry?.items ?? [{ projectId: '', hours: 0, description: '' }]
    },
    validators: {
      onChange: entrySchema
    }
  })

  const isEditable = !entry || entry.status === 'draft' || entry.status === 'rejected'
  
  if (!entry) {
    return (
      <div className="flex-1 flex items-center justify-center border rounded-lg">
        <Button onClick={() => createEntry.mutate({ date })}>
          Create Entry for {format(date, 'MMMM d')}
        </Button>
      </div>
    )
  }

  return (
    <div className="flex-1 border rounded-lg p-4">
      <div className="flex items-center justify-between mb-4">
        <div>
          <h2 className="text-lg font-semibold">
            {format(date, 'EEEE, MMMM d, yyyy')}
          </h2>
          <div className="flex items-center gap-2 mt-1">
            <StatusBadge status={entry.status} />
            <span className="text-sm text-muted-foreground">
              Total: {calculateTotal(entry.items)}h
            </span>
          </div>
        </div>
        {isEditable && (
          <Button variant="destructive" size="sm" onClick={() => deleteEntry.mutate(entry.id)}>
            Delete
          </Button>
        )}
      </div>

      <form onSubmit={form.handleSubmit(onSubmit)}>
        <div className="space-y-2">
          {form.field('items').map((item, index) => (
            <EntryRow
              key={index}
              item={item}
              index={index}
              editable={isEditable}
              onRemove={() => form.removeItem(index)}
            />
          ))}
        </div>

        {isEditable && (
          <Button variant="outline" size="sm" className="mt-3" onClick={addRow}>
            + Add Project Row
          </Button>
        )}

        {isEditable && (
          <div className="mt-4 flex gap-2">
            <Button type="submit" disabled={totalHours === 0}>
              Save Draft
            </Button>
            <Button 
              type="button"
              variant="default"
              disabled={totalHours === 0}
              onClick={() => submitEntry.mutate(entry.id)}
            >
              Submit Entry
            </Button>
          </div>
        )}
      </form>
    </div>
  )
}
```

## 10. Form Validation

### Zod Schema

```tsx
const entryItemSchema = z.object({
  projectId: z.string().uuid('Select a project'),
  hours: z.number().min(0).max(24),
  description: z.string().optional(),
})

const entrySchema = z.object({
  items: z.array(entryItemSchema)
    .min(1, 'At least one project row is required')
    .refine(
      (items) => items.reduce((sum, i) => sum + i.hours, 0) <= 24,
      'Total hours cannot exceed 24 per day'
    ),
})
```

### TanStack Form Integration

```tsx
const form = useForm({
  defaultValues: {
    items: entry?.items ?? [{ projectId: '', hours: 0, description: '' }]
  },
  validators: {
    onChange: entrySchema
  },
  onSubmit: async ({ value }) => {
    if (entry) {
      await updateEntry.mutateAsync({ id: entry.id, ...value })
    } else {
      await createEntry.mutateAsync({ date, ...value })
    }
  }
})
```

```tsx
export function useTimeEntry(date: Date) {
  return useQuery({
    queryKey: ['time-entries', format(date, 'yyyy-MM-dd')],

export function useProjects() {
  return useQuery({
    queryKey: ['projects'],
```

### Mutation Hooks

```tsx
export function useCreateTimeEntry() {
  const queryClient = useQueryClient()
  
  return useMutation({
    mutationFn: (data: CreateEntryRequest) =>
      api<TimeEntry>('/time-entries', {
        method: 'POST',
        body: JSON.stringify(data),
      }),
    onSuccess: (_, variables) => {
      queryClient.invalidateQueries({ queryKey: ['time-entries'] })
    },
  })
}

export function useUpdateTimeEntry() {
  const queryClient = useQueryClient()
  
  return useMutation({
    mutationFn: ({ id, ...data }: UpdateEntryRequest) =>
      api<TimeEntry>(`/time-entries/${id}`, {
        method: 'PUT',
        body: JSON.stringify(data),
      }),
    onSuccess: () => {
      queryClient.invalidateQueries({ queryKey: ['time-entries'] })
    },
  })
}

export function useDeleteTimeEntry() {
  const queryClient = useQueryClient()
  
  return useMutation({
    mutationFn: (id: string) =>
      api(`/time-entries/${id}`, { method: 'DELETE' }),
    onSuccess: () => {
      queryClient.invalidateQueries({ queryKey: ['time-entries'] })
    },
  })
}

export function useSubmitEntry() {
  const queryClient = useQueryClient()
  
  return useMutation({
    mutationFn: (id: string) =>
      api<TimeEntry>(`/time-entries/${id}/submit`, { method: 'POST' }),
    onSuccess: () => {
      queryClient.invalidateQueries({ queryKey: ['time-entries'] })
      toast.success('Entry submitted for approval')
    },
  })
}

export function useSubmitMonth() {
  const queryClient = useQueryClient()
  
  return useMutation({
    mutationFn: ({ month, year }: { month: number; year: number }) =>
      api(`/time-entries/submit-month?month=${month}&year=${year}`, {
        method: 'POST',
      }),
    onSuccess: () => {
      queryClient.invalidateQueries({ queryKey: ['time-entries'] })
      toast.success('All drafts submitted')
    },
  })
}
```

## 17. Implementation Guidelines

### Performance Best Practices

Reference Vercel React Best Practices during implementation:

| Priority | Category | Key Rules |
|----------|----------|-----------|
| CRITICAL | Waterfalls | `async-parallel` - Use Promise.all for independent fetches |
| CRITICAL | Bundle | `bundle-barrel-imports` - Import directly, avoid barrel files |
| MEDIUM-HIGH | Client Data | TanStack Query handles deduplication automatically |
| MEDIUM | Re-render | `rerender-memo` - Memoize expensive components |
| MEDIUM | Rendering | `rendering-conditional-render` - Use ternary, not && |

### Component Libraries

Use Shadcn/UI components via MCP during implementation:
- Initialize project with `shadcn init`
- Add components: button, input, select, dialog, toast, dropdown-menu, calendar
- Use `@tanstack/react-query` for data fetching with built-in deduplication
